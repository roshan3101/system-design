# Structural Patterns

Patterns focused on object composition and interfaces:
- Adapter (unify incompatible interfaces)
- Bridge (separate abstraction from implementation)
- Composite (tree structures treated uniformly)
- Decorator (add behavior without subclassing)
- Facade (simplified interface to subsystems)
- Flyweight (share intrinsic state to save memory)
- Proxy (stand-in for real object with access/control)

Each subfolder includes an example with its own `README.md` and `main.cpp`. Build with `g++ -std=c++17 -Wall -Wextra -o demo main.cpp`.
